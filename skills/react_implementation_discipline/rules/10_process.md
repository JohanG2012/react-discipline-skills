# Implementation Process

## Summary
Defines the disciplined workflow for applying changes.

---

## Rule: Plan-Driven Implementation
**Rule ID:** rid-process  
**Priority:** MUST  
**Applies to:** react_implementation_discipline  
**Rationale:** Ensures changes align with the approved plan.

### Requirement

- Follow the file touch plan and layer justification.
- Apply minimal changes required to satisfy the request.
- Validate boundary rules after changes.

### Forbidden

- Editing files outside the approved plan.
- Introducing new dependencies without approval.

### Notes

- Use deterministic defaults when implementation choices are unclear.
